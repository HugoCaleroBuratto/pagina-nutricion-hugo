## Cambio de obejetivo de la aplicacion

La aplicacion debe pasar a ser una app con la cual se pueda cargar un plan de alimentacion establecido o armar uno mismo. Para ello debe existir la posibilidad de generar su propia dieta o cargar algunos de los planes que ya van a estar pre establecidos.`Se debe poder cargar los planes ya armados para que las personas puedan elegir cual prefieren.`.

Se va a brindar una calculadora de calorias con la cual el usuario va a poder poner las variables de sus caracteristicas y que objetivo quiere. A partir de eso le dira el total de calorias y junto a ello podra elegir el plan que mejor se adapte. 

Los planes ya existentes van a ser de:
- 1500 cals
- 2000 cals
- 2500 cals
- 3000 cals
- 3500 cals

Una vez que se seleccione el plan, se carga la dieta pre establecida con el dia a dia de lo que se tiene que comer. Debe existir una ventana que indique las cantidad totales del mes para facilitar la compra de los productos y por otro lado un informe diario en el cual el usuario deberia de poder interactuar.

### Interaccion del usuario

El usuario va a poder completar dia a dia si pudo cumplir o no con la dieta y la dificultad que se le presento. Para ello, debe de tener las opciones de las distintas comida y que pueda marcar con un boton o una casilla de verificacion si lo pudo cumplir o no.\

A partir de esto, se debera de poder armar un informque que permita saber que porcentaje de cumplimiento tuvo el usuario.

## Carga de plan

Se debe brindar informacion nutricional de multiples alimentos para que el usuario pueda generar su propio plan a partir de una dieta que le dio un instructor. Con esto se deberia de poder cargar un plan diario y que se haga la equivalencia de alimentos para todo el mes.

La idea actual no es mala pero necesita tener una vuelta de rosca.

## Informe

Se debera de poder ver un informe mensual que permita ver estadistica de como fue el proceso en el mes.
